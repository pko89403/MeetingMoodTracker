🚀 Copilot CLI 하네스 구축 히스토리 리포트

본 리포트는 MeetingMoodTracker 모노레포(FastAPI + React) 환경에 GitHub Copilot CLI 기반의 자율 에이전틱 워크플로우(Agentic Workflow) 하네스를 구축한 전체 작업 과정과 결과물을 상세히 기록합니다.

⏱ 작업 타임라인 (하네스 구축 과정)

Step 1. 초기 전략 수립 및 워크트리 생성

기존 프로젝트의 안전한 변경을 위해 branch-pr-workflow 스킬을 활용하여 새로운 Git 워크트리(Worktree)와 격리된 브랜치(codex/copilot-harness)를 생성했습니다.

  • 경로: ~/.codex/worktrees/copilot-cli-harness/MeetingMoodTracker
  • 가상환경 분리 및 의존성 설치 스크립트 실행.
Step 2. 1차 시도 (커스텀 스크립트 작성) 폐기됨

처음에는 로컬 검증 도구(agent_runner.py)를 단순 래핑하는 커스텀 셸 스크립트(run-linter.sh, run-tests.sh)를 자체 제작하여 연동하려 했습니다.

결과: 하지만 사용자 피드백을 통해 "직접 만드는 것보다 이미 외부(커뮤니티)에 잘 구현된 표준 규격을 가져오는 것이 맞다"는 판단을 내렸고, 작성했던 가짜(Custom) 폴더 구조를 모두 삭제(rm -rf) 및 폐기했습니다.

Step 3. 공식 리포지토리 동기화 (Awesome-Copilot)

가장 최신의 검증된 표준을 적용하기 위해, 임시 경로에 github/awesome-copilot 공식 리포지토리를 클론(Clone)하고 내부의 600여 개 리소스 구조를 분석했습니다.

git clone https://github.com/github/awesome-copilot.git
Step 4. 최종 하네스 이식 및 적용 완료

공식 리포지토리에서 MeetingMoodTracker의 기술 스택(Python/FastAPI, React/Vite)에 완벽히 호환되는 실제 도구들만 엄선하여 추출(Copy)한 뒤, 프로젝트 루트의 .github/ 폴더에 공식 6단계 규격(Agents, Instructions, Skills, Hooks, Workflows, Plugins)으로 재배치했습니다.

📂 최종 적용된 공식 리소스 명세 (The Harness)

현재 워크트리에 성공적으로 이식되어 git add된 .github/ 내부 구조입니다. 이번엔 워크플로우(Workflows)와 플러그인(Plugins) 생태계 연동도 포함되었습니다.

🤖 Agents (에이전트)

Copilot 세션에서 특정 지식과 권한을 가진 전문가 페르소나를 호출합니다.

  • .github/agents/api-architect.agent.md: 백엔드 OpenAPI 및 시스템 통합 아키텍트.
  • .github/agents/expert-react-frontend-engineer.agent.md: React UI/UX 및 상태관리 최적화 엔지니어.
  • .github/agents/python-mcp-expert.agent.md: Python 환경의 Model Context Protocol(MCP) 서버 구축 전문가.
📋 Instructions (지침서)

파일 확장자에 따라 백그라운드에서 자동 로드되는 코딩 표준 규칙입니다.

  • .github/instructions/python.instructions.md: Python 파일 작성 시 FastAPI/Pydantic 규칙 준수.
  • .github/instructions/vitest.instructions.md: 프론트엔드 TypeScript 작성 시 Vite 생태계 및 테스트 원칙 준수.
  • .github/instructions/code-review.instructions.md: PR 및 코드 리뷰 시 범용 보안/OWASP 기준 점검.
🎯 Skills (스킬)

AI 에이전트가 코드를 교정하거나, 사용자가 /skill run [이름]으로 실행할 수 있는 검증 도구입니다.

  • .github/skills/ruff-recursive-fix/: Python 코드의 오류와 포맷팅을 일괄 수정합니다.
  • .github/skills/pytest-coverage/: 백엔드 테스트를 실행하고 커버리지 리포트를 생성합니다.
  • .github/skills/javascript-typescript-jest/: 프론트엔드 컴포넌트의 테스트 환경을 연동합니다.
  • .github/skills/playwright-generate-test/: E2E 테스트 시나리오 초안을 자동 생성합니다.
  • .github/skills/conventional-commit/: 표준화된 양식으로 커밋 메시지를 작성합니다.
🪝 Hooks (훅)

Copilot CLI 세션에서 보안과 규정 준수를 위해 백그라운드에서 자동 실행됩니다.

  • .github/hooks/secrets-scanner/: 소스 코드나 프롬프트에 하드코딩된 API Key 등 민감 정보 유출을 차단합니다.
  • .github/hooks/dependency-license-checker/: 새로 추가되는 라이브러리의 라이선스 충돌 여부를 검사합니다.
  • .github/hooks/session-logger/: 감사(Audit) 목적으로 AI와의 대화/프롬프트 내역을 안전하게 로깅합니다.
⚡ Agentic Workflows (워크플로우) New

Markdown 기반으로 작성되어, GitHub Actions 등 CI 파이프라인에서 AI가 주도적으로 수행하는 자동화 프로세스입니다.

  • .github/workflows/daily-issues-report.md: 매일 자정에 저장소의 이슈들을 요약 및 분석하여 보고서를 자동 생성하는 워크플로우.
  • .github/workflows/ospo-release-compliance-checker.md: 릴리즈(배포) 전 오픈소스 컴플라이언스(라이선스 등)를 AI가 검수하는 워크플로우.
🔌 Plugins (플러그인 생태계) New

개별 파일들을 일일이 복사할 필요 없이, 도메인별 Agent/Skill 번들을 Copilot CLI 환경에서 명령어로 즉시 설치할 수 있는 생태계입니다.

  • FastAPI 플러그인: copilot plugin install openapi-to-application-python-fastapi@awesome-copilot
    → OpenAPI 명세서로 FastAPI 애플리케이션 코드를 자동 생성하고 검증하는 번들
  • React 프론트엔드 플러그인: copilot plugin install frontend-web-dev@awesome-copilot
    → React/TypeScript/Vite 전문가 에이전트와 Playwright 테스트 스킬 번들
  • 플러그인은 로컬 폴더에 묶이지 않고, CLI 설치 명령어를 통해 필요할 때마다 전역/로컬로 주입됩니다.

이 문서는 개발자가 전체 하네스 구축 과정을 추적하고 검증하기 위해 동적으로 생성되었습니다.
