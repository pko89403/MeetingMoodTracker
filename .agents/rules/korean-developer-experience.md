# Rule: 한국어 우선 개발자 경험 (Korean Developer Experience)

MeetingMoodTracker는 한국어 기반 팀 협업을 전제로 합니다. 에이전트는 아래 원칙을 기본 행동으로 따릅니다.

## 1) 문서/가이드
- AGENTS, README, `docs/` 내 운영/설계 문서는 한국어를 기본 언어로 유지합니다.
- 외부 서비스 명칭, SDK 함수명 등 고유 영어 식별자는 그대로 사용하되, 필요한 경우 한국어 설명을 바로 붙입니다.

## 2) 코드 문서화
- `app/`, `scripts/`의 클래스/함수는 한국어 docstring을 작성합니다.
- docstring에는 최소한 "무엇을 하는지"가 드러나야 하며, 모호한 한 줄(예: "처리한다")만 단독으로 쓰지 않습니다.

## 3) API/운영 메시지
- 사용자/운영자가 보게 되는 오류 설명은 한국어 기준으로 해석 가능한 형태를 유지합니다.
- 외부 정책 오류(예: Azure content filter, VNet 정책)는 원문 코드를 보존하되 한국어 요약을 함께 제공합니다.

## 4) 테스트 데이터
- 회의 발화 데이터는 한국어 케이스를 우선 포함합니다.
- 한/영 혼합 발화(code-switching) 케이스를 반드시 최소 1개 이상 유지합니다.

## 5) 문서 동기화
- 한국어 DX 관련 동작/정책을 바꾸면 `AGENTS.md`와 관련 `docs/`를 동시에 갱신합니다.
