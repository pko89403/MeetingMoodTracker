작업 도중이거나 작업이 완료되어 사용자에게 결과를 선고하기 직전, 하네스(Pre-commit Hook)가 동작하여 에러를 뿜는 일(exit 2)을 미연에 방지하려면 이 워크플로우를 직접 기계적으로 수행하십시오.

1. **포매팅 픽스**: `uv run ruff format app/ tests/` 를 터미널에서 실행하여 공백이나 콤마 등을 포매터로 자동 교정합니다.
2. **린터 자동 수정**: `uv run ruff check --fix app/ tests/` 를 실행하여 사용하지 않는 `import` 찌꺼기나 에러들을 강제로 교정합니다.
3. **단위 테스트 및 아키텍처 체커 가동**: `uv run pytest` 명령을 터미널에서 실행합니다. `test_imports.py`에서 AST 헌법이 위반되어 붉은 줄이 출력되는지 않은지와 기존 개발된 비즈니스 테스트 들이 모두 `PASSED`인지 확인합니다.
4. 모든 항목에서 초록불이 들어왔다면 안심하고 작업을 종결 짓고 사용자에게 결과를 브리핑합니다.
