---
description: 새로운 API 엔드포인트를 명세에 맞추어 생성하는 워크플로우
---

새로운 비즈니스 로직이나 API 라우터를 생성하라는 복잡한 지시를 받았을 때 이 워크플로우 시퀀스에 맞춰 순차적으로 구체화하십시오.

1. **스키마 정의 확정**: `app/types/` 내부에 해당 기능의 입출력을 담당할 Pydantic Request/Response 스키마 모델을 가장 먼저 설계해야 합니다.
2. **비즈니스 로직 구현**: `app/service/` 계층에 스키마를 주고받아 연산을 처리하거나 도구 호출을 수행할 핵심 로직 단위 클래스/함수를 작성합니다. 이 때 `runtime` 등을 절대 참조하지 않게 조심합니다.
3. **엔드포인트 연결**: `app/runtime/` 계층에 라우터(`/api/v1/...`)를 개설하고 생성한 서비스 로직을 라우터 컨트롤러에 바인딩합니다.
4. **유닛 테스트 확인**: 기 작성된 코드를 테스트하기 위해 `tests/` 디렉터리에 HTTP 테스트 클라이언트 `TestClient`를 이용한 동작 검증 코드를 작성합니다.
